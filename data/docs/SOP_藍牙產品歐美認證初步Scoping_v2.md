# 藍牙產品歐美認證初步 Scoping SOP

版本：v2.0  
適用對象：認證工程師、專案管理、業務與客服窗口  
文件目的：協助針對藍牙產品進行歐盟與美國市場的初步認證範圍判斷。

## 1. Scoping 目的

本 SOP 用於初步整理藍牙產品進入歐盟與美國市場時常見的認證評估項目。此文件僅供初步 scoping 使用，不構成最終認證結論。

正式判定仍需依產品完整規格、銷售地區、無線模組狀態、天線設計、使用場景與既有測試報告，由認證專員確認。

## 2. 歐盟市場初步評估

### 2.1 RED 指令

若產品具備藍牙、Wi-Fi、Zigbee、NFC 或其他無線發射功能，銷售至歐盟時通常需評估 Radio Equipment Directive，簡稱 RED。

初步需確認：

- 產品是否具備無線發射功能。
- 使用的無線技術與頻段。
- 輸出功率與天線型式。
- 是否使用已認證模組。
- 模組是否依原條件整合。
- 產品最終用途與使用環境。
- 是否已有測試報告、DoC 或技術文件。

### 2.2 常見補充資料

進行 RED 初步 scoping 時，建議向客戶取得：

- 產品型號與產品照片。
- 電路方塊圖與使用者手冊。
- 無線模組規格書。
- 天線規格與安裝位置。
- 預計銷售國家。
- 既有 CE、RED 或 EMC 測試報告。

## 3. 美國市場初步評估

### 3.1 FCC Part 15

若藍牙產品銷售至美國，通常需評估 FCC Part 15 相關要求。若產品使用已取得 FCC ID 的模組，仍需確認最終產品整合方式是否符合模組授權條件。

初步需確認：

- 是否使用已認證模組。
- FCC ID 是否可清楚標示。
- 天線是否與模組認證條件一致。
- 是否改變輸出功率、頻段或韌體設定。
- 最終產品是否可能需要額外 EMC 或 RF 測試。

### 3.2 可能需要轉人工的情境

以下情境應轉由認證專員確認：

- 客戶要求保證 FCC 一定通過。
- 模組、天線或韌體條件與既有認證不一致。
- 客戶提供的資料不足以判斷測試範圍。
- 涉及正式報價、交期或合約承諾。
- 客戶詢問是否可以規避測試或省略必要資料。

## 4. 初步回覆模板

「依目前提供資訊初步判斷，此藍牙產品若銷售至歐盟，通常需評估 RED 指令相關要求；若銷售至美國，通常需評估 FCC Part 15。實際測試範圍仍需依產品規格、無線模組、天線設計、銷售地區與既有報告確認。」

## 5. 注意事項

AI 或客服窗口不得直接承諾產品一定通過認證，也不得對外給出最終法規結論。若問題涉及保證、報價、交期、法律責任或系統變更，應依高風險轉人工流程處理。
